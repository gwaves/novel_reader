# 离线多角色 TTS 开发计划

## Phase 0：目录与文档

状态：已完成。

- 新建 `offline-tts/`。
- 确定主流程使用 Node.js。
- 明确 Codex 不参与批量大模型推理。
- 增加示例配置、设计文档、开发计划和 CLI 脚本。

## Phase 1：导演脚本原型

目标：对短章节片段生成可检查的导演脚本 JSON。

任务：

- 从 `~/.novel_reader/novel_reader.sqlite` 读取书籍和章节。
- 对章节正文做规则预切分，确保引号内对白和旁白分离。
- 从配置读取角色与音色绑定。
- 从知识图谱读取候选人物和别名。
- 调用配置的大模型生成 `decisions` JSON。
- 由程序回填原文，组装最终导演脚本。
- 输出诊断文件和校验结果。

验证目标：

- 使用《妖刀记》第 1 章前 1500 到 3000 字。
- 旁白必须保持男声。
- 引号内对白不能和旁白混在同一段。
- 能识别已有证据支持的 `采蓝`、`黄缨`。
- 不确定的对白必须标为 `未知角色`，不能强猜。

## Phase 2：脚本校验与修复

目标：让导演脚本足够安全，可以进入批处理。

任务：

- 完善 `validate-script` 命令。
- 增加原文覆盖率检查。
- 增加确定性修复能力，例如重编号、旁白音色重置、未知角色置信度修正。
- 输出低置信度片段报告，供人工复核。
- 形成稳定输出目录。

建议目录：

```text
tmp/tts/<book-safe-name>/ch001/
  director-script.json
  director-script.diagnostics.json
```

## Phase 3：MIMO TTS 合成

状态：已开始。

目标：从导演脚本合成分段音频。

任务：

- 增加 `synth` 命令。已完成初版。
- 按 segment 解析音色和风格。已完成初版。
- 用 MIMO TTS 合成每个片段。已完成初版，运行时需要 `MIMO_API_KEY`。
- 按 segment ID 和文本 hash 缓存 WAV。已完成初版。
- 增加瞬态失败重试。
- 输出分段 manifest。已完成初版。

## Phase 4：音频拼接与 MP3 编码

状态：已开始。

目标：生成完整章节音频，同时控制磁盘占用。

任务：

- 把分段 WAV 标准化为统一采样率和声道。已完成初版。
- 在旁白和对白切换处插入可配置停顿。已完成初版。
- 使用 `ffmpeg` concat 拼接。已完成初版。
- 默认把最终产物编码为 MP3，例如 `96k`。已完成初版。
- 标准化 WAV 默认只作为临时文件。
- 输出 `chapter.mp3`、`manifest.json` 和可选 `srt`。

## Phase 5：批处理

目标：支持章节范围或整本书处理。

任务：

- 增加任务队列和断点续跑。
- 分别控制 LLM 并发和 TTS 并发。
- 输出耗时、进度和失败原因。
- 支持失败片段重试。
- 支持仅生成导演脚本、不合成音频的阶段性执行。

## Phase 6：音色设计实验

目标：验证 MIMO `mimo-v2.5-tts-voicedesign` 能否改善角色年龄感和声线贴合度。

任务：

- 为角色配置增加 `voiceMode: "preset" | "design"`。
- 为角色配置增加 `voiceDesignPrompt`。
- 抽取采蓝、黄缨各 2 到 3 个片段生成 A/B 样本。
- 对比预置音色和音色设计的年龄感、角色感、跨片段一致性。
- 如果一致性可接受，再考虑让整章合成支持按角色选择 preset 或 design。
