# 产品功能说明书

更新时间：2026-06-30

本文档用于把小说阅读助手从快速开发阶段沉淀为可测试、可评审、可运维的产品规格。后续测试用例、系统性 Code Review、监控与发布检查都应以本文档为功能基线。

## 1. 产品定位

小说阅读助手是一个本地优先的长篇小说阅读、知识整理和移动阅读系统。PC 端负责导入、阅读、AI 概要、知识图谱、RAG 检索和内容生产；Gateway 负责向移动端分发书库、数据包、音频、RAG 能力和运维后台；Gateway Android App 负责移动端离线阅读、缓存 MP3 和应用更新。

## 2. 用户角色

| 角色 | 主要目标 | 主要界面 |
|------|----------|----------|
| 阅读用户 | 导入小说、阅读、恢复进度、搜索剧情、查看概要和图谱 | PC Web、Gateway Android App |
| 内容生产者 | 批量生成概要、图谱、embedding、音频和移动数据包 | PC Web、production-pipeline、offline-tts |
| 运维/管理员 | 管理 Gateway 书库、数据包、音频、设备授权、请求日志和下载发布 | Gateway Admin UI |
| 开发/测试人员 | 验证功能契约、回归测试、发布检查和问题定位 | 文档、测试套件、日志、指标接口 |

## 3. 系统边界

### 3.1 PC 本地阅读器

- 导入 `.txt` 和 `.epub` 小说。
- 自动拆章，展示书架、章节目录和正文。
- 保存每本书阅读进度，支持章节跳转、键盘/按钮翻页和阅读偏好。
- 提供模型配置、概要生成、知识图谱、RAG 搜索和数据库备份恢复。
- 数据默认保存在本机 SQLite：`~/.novel_reader/novel_reader.sqlite`。

### 3.2 AI 与知识处理

- 支持本地 Ollama 与 OpenAI-compatible 模型配置。
- 生成模型和 embedding 模型独立配置、独立校验。
- 概要支持单章、当前页和全书缺失章节批量生成。
- 知识图谱支持实体、关系、证据、低置信度复审、实体合并/拆分、关系维护、共指合并、重扫预览和 JSON/GraphML 导出。
- RAG 搜索融合章节概要向量、正文 chunk 向量和图谱实体召回，并生成带来源依据的回答。

### 3.3 离线与生产流水线

- `scripts/offline-scanner.mjs` 负责浏览器外批量概要/图谱扫描、续扫、导入导出和数据包生成。
- `production-pipeline/` 是正式内容生产入口，目标流程为 `import -> summary -> kg -> embedding -> audio -> package -> publish -> verify`。
- `offline-tts/` 负责多角色导演脚本生成、TTS 合成和章节 MP3 输出。

### 3.4 Gateway 服务

- Gateway 是移动端和运维后台的服务边界，不直接把本地 SQLite API 暴露到公网。
- 移动端接口提供书库、单书 package、音频目录、受保护音频下载、RAG 搜索和 RAG 答案。
- 管理端接口提供书籍、数据包、音频、设备、指标、事件、请求日志和下载发布能力。
- 生产环境应使用独立 admin/mobile token，并保留设备角色和可见范围控制。

### 3.5 Gateway Admin UI

- 管理后台面向内网/运维使用。
- 核心视图包括总览、书籍、数据包、音频、设备、请求日志和设置。
- 总览应展示真实运行指标、内容健康、下载趋势、最近事件和设备摘要，不应在真实接口可用时回退 mock 数据。
- 操作类功能应具备保存中、成功、失败回滚、重试和确认提示。

### 3.6 Gateway Android App

- 通过 Gateway 连接书库和单书 package。
- 支持本地缓存书籍、阅读进度、阅读偏好和章节 MP3。
- 支持离线阅读与本地缓存 MP3 播放。
- 根据设备角色处理 default/trusted/disabled 状态。
- 支持应用内检查 Gateway 发布的 APK 更新，并交由 Android 系统确认安装。

### 3.7 历史移动端

旧 `mobile-app/` 局域网同步客户端已进入退役状态，仅保留历史参考。新移动端功能默认在 `gateway-android-app/` 中维护。

## 4. 主流程规格

### 4.1 本地导入与阅读

1. 用户在 PC Web 选择 `.txt` 或 `.epub`。
2. 系统解析章节并写入本地 SQLite。
3. 用户进入阅读器，选择章节并阅读。
4. 系统按书保存章节、滚动位置和阅读偏好。
5. 用户重新打开应用时恢复最近阅读状态。

验收重点：

- txt 编码、章节标题和超长章节拆分稳定。
- epub spine 顺序正确，正文可读。
- 多本书阅读进度互不覆盖。
- 数据库备份恢复后书架、章节、概要、图谱和设置完整。

### 4.2 概要生成

1. 用户配置生成模型。
2. 用户选择单章、当前页或全书缺失章节。
3. 系统调用模型并保存概要。
4. 失败章节不应中断整批任务。

验收重点：

- 超过阈值的批量任务有确认。
- 并发、失败统计和重试提示清晰。
- 已生成概要不被误覆盖，除非用户明确触发重扫/覆盖。

### 4.3 知识图谱

1. 用户选择当前章节、章节范围或全书扫描。
2. 系统保存章节级 raw extraction，并归一化实体和关系。
3. 用户通过列表、详情、复审队列和图谱视图检查结果。
4. 用户可编辑、合并、拆分、删除实体或关系。

验收重点：

- 扫描任务可恢复、可停止、可跳过已完成章节。
- 高风险操作有可追踪记录，后续应补齐回滚能力。
- 共指合并、覆盖重扫和批量删除必须进入回归测试矩阵。

### 4.4 RAG 搜索

1. 用户配置 embedding 模型并生成覆盖率。
2. 用户输入跨章节问题。
3. 系统召回概要、正文 chunk 和图谱实体。
4. 用户可查看来源，并生成回答。

验收重点：

- embedding 覆盖率不足时给出明确阻断提示。
- 搜索结果可跳转章节或图谱详情。
- 回答必须基于召回上下文，不应隐藏检索失败。

### 4.5 内容生产与发布

1. 内容生产者选择 job JSON。
2. production-pipeline 执行导入、概要、图谱、embedding、音频、package、发布、验证。
3. Gateway 接收 package、音频和 APK 下载文件。
4. 管理后台显示真实覆盖率、音频健康和发布状态。

验收重点：

- 每个阶段可独立运行、可恢复、可观察。
- publish/verify 不应只信本地日志，必须检查 Gateway 实际可见结果。
- package、audio 和 APK 发布元数据要可追踪。

### 4.6 Gateway 移动阅读

1. Android App 配置 Gateway 地址、token 和设备名。
2. App 验证会话并同步书库。
3. 用户选择书籍，下载 package 并阅读。
4. 用户可缓存章节 MP3，离线播放。
5. App 根据 Gateway 发布元数据检查更新。

验收重点：

- 禁用设备不能继续访问云端资源，但本地缓存阅读应有明确策略。
- 阅读进度按书保存，切书、切 Tab、重启后不丢失。
- MP3 缓存状态按书/章节隔离，不串书。
- 更新检查只能提示安装，最终安装必须由 Android 系统确认。

## 5. 非功能规格

| 类别 | 要求 |
|------|------|
| 安全 | admin/mobile 鉴权分离；公网不暴露 dev fallback；移动端不保存上游模型密钥 |
| 可测试性 | 每个主流程需要单元、接口、UI 或手工验收用例；高风险数据操作必须有回归用例 |
| 可观测性 | Gateway、production-pipeline 和发布流程应提供指标、事件、日志和失败原因 |
| 可恢复性 | 批处理任务可续跑；数据库恢复前自动备份；高风险图谱操作后续需补审计和回滚 |
| 性能 | 大型 package 解析需缓存；移动端大 MP3 下载避免 JS/Blob 内存膨胀 |
| 兼容性 | Android WebView、Capacitor 原生插件、HTTP cleartext 开发场景和本地浏览器都需覆盖 |
| 文档 | README 说明主流程；规格、测试、运维文档作为后续开发入口 |

## 6. 后续测试设计入口

测试用例应按以下维度展开：

- 功能域：PC 阅读、AI 概要、知识图谱、RAG、生产流水线、Gateway、Admin UI、Android App。
- 测试层级：单元、接口、集成、浏览器 UI、Android 真机/模拟器、生产发布验收。
- 风险等级：数据破坏、鉴权越权、发布失败、缓存串书、进度丢失、模型调用失败。
- 可观测性：每个失败场景是否有可定位的日志、事件、指标或用户提示。

详细执行路线见 [正规化测试与运维规划](quality-ops-roadmap.md)。
